================================================================================
TEST 1: Register a new user
================================================================================
curl -X POST http://localhost:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name":"Lucas","email":"lucas@test.com","password":"senha123","user_type":"usuario"}'

EXPECTED: {"message": "User created!", "user_id": 1}

================================================================================
TEST 2: Register duplicate email (should fail)
================================================================================
curl -X POST http://localhost:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name":"Lucas2","email":"lucas@test.com","password":"senha456","user_type":"usuario"}'

EXPECTED: {"detail": "Email already registered"} (400 error)

TEST 9: Create a chamado as regular user (should succeed)
================================================================================
OPTION A: Simple JSON (Recommended for testing)
curl -X POST http://localhost:8000/chamados/json \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"title":"Problema com impressora","description":"A impressora não está funcionando"}'

OPTION B: Multipart/form-data (use if uploading files)
curl -X POST http://localhost:8000/chamados \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -F 'chamado={"title":"Problema com impressora","description":"A impressora não está funcionando"};type=application/json'

With files (Option B only):
curl -X POST http://localhost:8000/chamados \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -F 'chamado={"title":"Problema com impressora","description":"A impressora não está funcionando"};type=application/json' \
  -F "files=@/path/to/file1.png" \
  -F "files=@/path/to/file2.pdf"

EXPECTED (both options): {"message": "Chamado criado", "id": 1}

Thunder Client / Swagger (Option A - JSON):
- Method: POST
- URL: http://localhost:8000/chamados/json
- Authorization: Bearer <YOUR_TOKEN>
- Body: JSON → paste the JSON above
- Send

USING THUNDER CLIENT (GUI - EASIER ALTERNATIVE TO CURL)
================================================================================
If you prefer a GUI instead of curl:

1. Open VS Code
2. Go to Extensions → Search "Thunder Client"
3. Click Install
4. Click Thunder Client icon on left sidebar
5. Click "New Request"
6. Set method to "POST"
7. Set URL to "http://localhost:8000/auth/login"
8. Go to "Auth" tab → Select "Bearer Token"
9. Paste your token in the field
10. Go to "Body" tab → Select "JSON"
11. Paste the request JSON
12. Click "Send"

Same tests, but with a user-friendly interface.

================================================================================
TROUBLESHOOTING
================================================================================

Q: Token is invalid after restart?
A: Tokens expire based on ACESS_TOKEN_EXPIRE_MINUTES in config.py (default 10 min).
   Re-login to get a fresh token.

Q: Getting "Not authenticated" even with token?
A: Make sure you include "Bearer " prefix: Authorization: Bearer <token>

Q: Getting database error?
A: Make sure you created the tables in MySQL using the SQL schema file.

Q: 404 on chamado ID?
A: Make sure the chamado_id exists. Create one first using Test 9.
